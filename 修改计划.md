目标：先把 `dajilin.net` 从“内容导航站”升级成“内容获客 + AI 初筛 + 顾问转化”的商业化 v1，不先做重电商。

**阶段 0：先修工程底座**
范围：保证后续能稳定构建和部署。

1. 修复 `npm run typecheck` 当前错误（见下方错误清单）。
2. 统一部署模式：确认线上到底走静态站点还是 Node SSR。
3. 暂时收口未成熟模块：商城（shop）、支付、auth-api 如果暂不投产，在 tsconfig.json 中 exclude 避免影响主站构建。
4. 保留现有未提交改动，不做大范围重构。

交付结果：`typecheck` 和 `test` 通过，后续可持续发布。

**阶段 1：首页商业化改版**

   3-5 日行程、亲子游、长白山/延边路线。
4. `资源合作入驻`
   驿站、酒店、餐饮、景区、研学资源方。

首页 CTA 调整：

- 主按钮：`提交旅行/研学需求`
- 次按钮：`让 AI 先帮我规划`
- 首页中段增加“热门服务”区块。
- 首页底部增加“为什么通过大吉林咨询”信任区块。

**阶段 2：合作咨询页改成强表单页**
目标：把 [合作咨询](https://dajilin.net/cooperation/) 从说明页改成线索入口。

页面结构：

1. 顶部明确文案：
   `提交你的需求，我们帮你判断适合的吉林文旅服务与合作路径。`
2. 表单字段：
   - 身份类型：游客 / 学校 / 研学机构 / 企业 / 驿站资源方 / 景区或文旅资源方
   - 需求类型：工业研学 / 企业参访 / 331 自驾 / 行程定制 / 资源合作 / 其他
   - 出行或合作时间
   - 人数规模
   - 联系人
   - 手机/微信
   - 需求描述
3. 提交后：
   - 写入现有 n8n webhook 或本地接口
   - 展示“已收到，顾问将联系你”
   - 标记来源页面、模块、语言、时间

**阶段 3：工业研学产品化**
目标：把 [工业研学](https://dajilin.net/study-tours/) 从资源展示变成可咨询服务页。

新增 3 个服务卡：

1. `长春工业研学 1 日营`
   适合学校、研学机构；方向：一汽红旗 / 工业文明 / 城市产业认知。
2. `企业参访与政企考察`
   适合企业、商协会、政府机构；方向：汽车、轨道交通、卫星、化工、农业产业。
3. `工业文明主题定制线路`
   适合定制团队；支持 1-3 日组合方案。

每张卡包含：

- 适合对象
- 建议人数
- 建议时长
- 可选资源
- 交付内容
- CTA：`提交研学需求`

注意：涉及一汽、中车等企业时，文案要保持“资源说明/参访咨询”，避免承诺未确认的开放权限。

**阶段 4：331 自驾服务产品化**
目标：把 [自驾攻略](https://dajilin.net/self-drive/) 变成自驾线索入口。

新增 3 个服务卡：

1. `331 吉林段路线规划`
   适合散客、自驾家庭、摄影用户。
2. `驿站与沿线服务对接`
   适合车队、团体、自驾俱乐部。
3. `边境自驾 / 延边深度路线`
   适合 3-5 日深度游、亲子、摄影团。

每张卡包含：

- 推荐时长
- 适合人群
- 服务内容
- 需要提前确认的信息
- CTA：`提交自驾需求`

**阶段 5：AI 助手转化增强**
目标：AI 不只是回答问题，而是帮平台做初筛。

调整方向：

1. 根据页面模块生成不同引导：
   - 工业研学页：问团队人数、年龄、时间、预算
   - 自驾页：问天数、车辆数、出发地、偏好
   - 目的地页：问季节、人数、亲子/摄影/休闲偏好
2. AI 回答后追加行动建议：
   - `生成初步路线`
   - `提交给顾问`
   - `查看相关服务`
3. 人工转接表单增加隐藏字段：
   - `source_page`
   - `module`
   - `user_question`
   - `locale`
4. 推荐问题从泛问题改成转化问题：
   - `我们学校 40 人想做长春工业研学，怎么安排？`
   - `331 自驾 4 天，想走延边和边境线，怎么规划？`
   - `第一次去吉林 5 天，长白山和延边怎么组合？`

- 首页 5 秒内能看懂平台服务。

- `npm run typecheck`、`npm test` 通过。
- 线上无明显死链、无“待提供”联系方式。
