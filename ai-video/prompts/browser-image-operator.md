# Codex 网页端图片生成操作规范

目标：Codex 控制网页端 GPT Image 2，把 `image-web-tasks.json` 中的提示词逐条提交并下载图片。

执行前置条件：
- 用户已指定网页端入口，例如 ChatGPT 网页端。
- 用户已登录账号。
- 用户已确认允许 Codex 将旅游视频提示词提交到该网页。
- 不提交 API key、密码、支付信息或任何敏感资料。

执行步骤：
1. 打开网页端图像生成工具。
2. 确认当前模型或模式是 GPT Image 2 / 图像生成。
3. 从 `image-web-tasks.json` 读取第一条任务。
4. 将 `prompt` 完整粘贴到输入框。
5. 提交生成。
6. 等待图片生成完成。
7. 下载图片，保存到对应 run 目录，文件名使用 `targetFile`。
8. 重复执行下一条任务。
9. 运行 `npm run ai:qa -- <topic> --run <runId>` 验证图片是否齐全。

如果网页端无法稳定自动下载：
- 先让 Codex 完成提交和生成。
- 用户手动下载图片。
- 按 `targetFile` 命名后放入 run 目录。

停止条件：
- 页面要求输入密码、验证码、支付信息或二次验证。
- 生成过程提示内容政策拦截。
- 网页端切换到非图像生成模型且无法恢复。
